## 🚧 計画中

### conntrack ルーター対応（OpenWrt / ASUS ルーターモード / Ubiquiti UDM）

Linux の `nf_conntrack` 用共通パーサーを実装することで、Linux ベースの多くのルーターに対応できます：

| ルーター | ファームウェア変更 |
|---------|-----------------|
| ASUS（ルーターモード） | 不要 — 標準ファームウェアで `/proc/net/nf_conntrack` が読める |
| Ubiquiti UDM / UDM-Pro / UDM-SE | 不要 — SSH（鍵認証） |
| OpenWrt（Buffalo・TP-Link 等） | 必要（フラッシュ） |

**🙋 実機テスター募集中** — 実装の大半はハードウェアなしで進められますが、実機での検証だけはできません。これらのルーターをお持ちの方は [Issue を立てて](https://github.com/yo1t/widemap/issues)ください。

### 通信ブロック（conntrack 対応の後）

ブロックルールをルーターに書き込みます（Yamaha は SSH 経由の `ip filter`）。まずは手動承認モードのみ。自動ブロックは、実運用で誤検知率が十分低いと実証できるまで計画しません。

## ❌ 予定なし

- **RA Proxy 方式の IPv6 セッション追跡** — 一部の ISP の ONU 配下では IPv6 接続が切断されることを実機確認済み。IPv6 可視化を実現する場合は、ミラーポート + オプションのキャプチャノード方式を採用します。
- **クラウド/SaaS へのデータ収集** — Widemap はローカルファーストです。通信メタデータがネットワークの外に出ることはありません。この方針は変わりません。
- **インライン型のトラフィック介在** — Widemap はパッシブ監視に徹します。データパスに介在したり遅延を加えることはありません。
