⚙ 設定
一般設定

世界地図上でアークの起点となる場所です。
日本以外を選択した場合、グリニッジを中心とした標準的な世界地図になります。

メモが未記入のノードに対してバックグラウンドで ARP / mDNS / ポート調査を行い、メモを自動生成します（既存メモは上書きしません）。

🛡️ 脅威検出設定

外部の脅威インテリジェンスフィードからマルウェアC2/配布元IPリストをダウンロードし、NAT セッションと突合します。

🔔 Slack通知

脅威検出時に Slack DM で通知します。

🔗 Slack Appを作成する（chat:write, users:read スコープ必要）
💾 バックアップ・リストア
📡 データソース設定

宛先ホスト名を解決するためのデータソースを設定します。
有効にしたソースからのドメイン名が通信ログなどに表示されます。

EC2 上の dnsmasq クエリログをテールし、デバイスごとの DNS 解決結果を宛先ホスト名に反映します。
PTR 逆引きより優先され、data.meethue.com のような意味のある名前が表示されます。

Yamaha syslog の [INSPECT] エントリをリアルタイムで解析し、60秒ポーリングで取りこぼした短命 TCP セッションを補完します。
EC2 に届く Yamaha syslog ファイルのパスを指定してください。

Yamaha syslog の [DHCPD] エントリ（Allocates/Extends）をリアルタイムで解析し、IP→MAC マッピングを維持します。
送信元デバイスの特定精度が向上します。通常は [INSPECT] と同じパスを指定します。
